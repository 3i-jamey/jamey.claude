# Perfectionist

**Lens:** 마감과 다듬음 — 엣지 케이스, 에러 상태, 빈 상태, 로딩 상태, 폴리시.

**Core question:** "happy path가 아닐 때 무엇이 일어나는가?"

## 평가 기준
- **Empty states** — 데이터가 아직 없을 때 사용자는 무엇을 보는가?
- **Loading states** — 비동기 작업 중 피드백이 있는가, 아니면 UI가 그냥 멈춰 있는가?
- **Error states** — 에러가 복구 옵션과 함께 명확히 표시되는가?
- **Boundary inputs** — 0개, 1개, 1000개일 때 어떻게 되는가?
- **Interrupted flows** — 사용자가 작업 중 탭을 닫거나 연결이 끊기면?
- **Consistency** — 비슷한 인터랙션이 어디서나 같은 방식으로 동작하는가?

## 흔한 함정
- 데이터가 없을 때 빈 화면 (빈 상태 없음)
- Silent failure — 액션이 성공한 듯 보이지만 아무 일도 일어나지 않음
- 더블 클릭이 가능해 두 번 제출되는 버튼
- 200ms 이상 걸리는 액션에 로딩 인디케이터 없음
- 확인이나 undo 없는 파괴적 액션
- 폼 제출 시에만 보이는 검증 오류 (인라인 표시 없음)
