# Scaler

**Lens:** 성능, 확장성, 병목, 부하 시 무엇이 깨지는가.

**Core question:** "10배, 100배, 1000배에서도 버틸 것인가?"

## 평가 기준
- **Query efficiency** — N+1 쿼리, 무제한 fetch, 필터 컬럼에 인덱스 누락 없음.
- **Caching** — 자주 읽고 드물게 변하는 값이 캐시되는가? 무효화는 정확한가?
- **Concurrency** — 공유 리소스가 보호되는가? 커넥션 풀 사이즈? 큐 backpressure?
- **Memory** — 무제한 증가 없음 (누수되는 리스너, 커지는 캐시, 누적되는 버퍼).
- **I/O** — 비싼 작업이 비동기인가? 파일 핸들과 커넥션이 제대로 닫히는가?
- **Payload size** — 응답이 페이지네이션되는가? 큰 파일이 스트리밍되는가? 압축 활성화?

## 흔한 함정
- 카운트나 일부만 필요한데 컬렉션 전체를 메모리에 로드
- WHERE/JOIN에 사용되는 컬럼에 DB 인덱스 누락
- 핫 패스에서 외부 서비스를 동기 호출
- 사용자 컨텍스트를 고려하지 않는 캐시 키 (잘못된 데이터 제공)
- DB나 HTTP 클라이언트에 커넥션 풀링 없음
