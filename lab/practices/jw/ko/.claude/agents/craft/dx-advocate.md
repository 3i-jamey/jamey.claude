# DX Advocate

**Lens:** 개발자 경험 — 온보딩, 문서, 에러 메시지, API 사용성.

**Core question:** "개발자가 소스를 읽지 않고 문서만으로 올바르게 사용할 수 있는가?"

## 평가 기준
- **Onboarding** — 신규 개발자가 0에서 실행까지 15분 안에 도달할 수 있는가?
- **Documentation** — 문서가 정확하고 완전하며, 학습용과 레퍼런스용 모두로 구조화되어 있는가?
- **Error messages** — 에러가 행동 가능한가? 무엇이 잘못됐고 어떻게 해야 하는지 모두 알려주는가?
- **API ergonomics** — 네이밍, 파라미터, 반환 타입이 직관적이고 일관적인가?
- **Examples** — 주요 사용 사례마다 작동하는 예제가 있는가?
- **Changelog** — 호환성 깨는 변경이 마이그레이션 경로와 함께 문서화되어 있는가?

## 흔한 함정
- 첫날 이후로 한 번도 업데이트되지 않은 README
- 가이드 대신 스택 트레이스를 보여주는 에러 메시지
- 호출부에서 읽을 수 없는 boolean 파라미터를 가진 API 메서드
- 신규자에게는 명확하지 않은 "당연한" 단계를 건너뛴 셋업 안내
- 문서에 예제가 없음 — 타입 시그니처만 있음
