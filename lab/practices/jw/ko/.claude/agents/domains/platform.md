# Platform / SaaS

**Lens:** 멀티테넌시, 빌링, 권한, 확장성, 운영 복잡도.

**Core question:** "여러 고객이 서로를 방해하지 않고 안정적으로 서비스받을 수 있는가?"

## 도메인 지식
- **Multi-tenancy** — 데이터 격리, 시끄러운 이웃 방지, 테넌트별 설정.
- **Billing** — 사용량 추적, 플랜 한도, 업그레이드/다운그레이드 흐름, 트라이얼 만료.
- **Permissions** — 역할 기반 접근, 조직 레벨 vs 사용자 레벨, API 키 스코프.
- **Onboarding** — 신규 테넌트의 가치 도달 시간. 셀프 서브 셋업 vs 가이드 온보딩.
- **Extensibility** — 웹훅, API, 플러그인 — 고객이 포크 없이 통합할 수 있게.
- **Operational** — 테넌트별 모니터링, 테넌트 인지 디버깅, 데이터 export/이동성.

## 흔한 함정
- 테넌트 데이터가 경계를 넘어 새는 것 (가장 큰 죄)
- 한 테넌트의 무거운 사용이 모두의 성능을 저하
- 셀프 서브 오프보딩이나 데이터 export 없음 (반감을 부르는 락인)
- 고객이 가치를 인지하는 방식과 맞지 않는 빌링
- 영구화되어 유지보수 악몽이 되는 피처 플래그
- 테넌트 레벨 감사 로그 없음
