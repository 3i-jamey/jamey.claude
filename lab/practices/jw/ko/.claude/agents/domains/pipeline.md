# Data Pipeline

**Lens:** ETL/ELT, 데이터 흐름, idempotency, 스키마 진화, 관측성.

**Core question:** "이 데이터를 정확하고, 반복 가능하며, 복구 가능하게 처리할 수 있는가?"

## 도메인 지식
- **Idempotency** — 같은 작업을 두 번 실행해도 같은 결과를 내야 한다. 중복 없음, 손상 없음.
- **Schema evolution** — 새 필드, 제거된 필드, 타입 변경을 어떻게 다루는가? Forward/backward 호환?
- **Backfill** — 라이브 데이터에 영향 없이 과거 데이터를 재처리할 수 있는가?
- **Partitioning** — 효율적 처리와 쿼리를 위해 시간/테넌트/키로 파티셔닝.
- **Observability** — 행 수, 처리 시간, 에러율, 데이터 신선도 — 모두 가시화.
- **Failure handling** — Dead letter queue, 재시도 정책, 멈춘/실패한 작업에 대한 알림.

## 흔한 함정
- idempotency 없음 — 재실행이 중복 생성
- 다운스트림 소비자를 깨뜨리는 스키마 변경
- 데이터 신선도 모니터링 없음 — 오래된 데이터가 인지되지 않음
- 증분만 필요한데 전체 데이터셋 처리
- 커스텀 스크립트 없이는 재처리/백필 불가
- 검증 실패 시 조용한 데이터 손실
