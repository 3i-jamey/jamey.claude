# Mobile Application

**Lens:** 네이티브/크로스 플랫폼 개발, 디바이스 제약, 오프라인 우선, 앱 스토어 정책.

**Core question:** "네이티브로 느껴지고, 디바이스 제약을 존중하며, 상시 연결 없이도 동작하는가?"

## 도메인 지식
- **Platform choice** — 네이티브 (iOS/Android), 크로스 플랫폼 (Flutter, React Native), PWA? 각각 UX와 유지보수 트레이드오프.
- **Offline first** — 핵심 기능은 네트워크 없이도 동작. 연결되면 동기화.
- **Device constraints** — 배터리, 메모리, 스토리지, 화면 크기, 입력 방식.
- **App lifecycle** — 백그라운드/포그라운드 전환, 푸시 알림, 딥링크.
- **Distribution** — 앱 스토어 리뷰 가이드라인, 업데이트 전략, 버저닝.
- **Permissions** — 필요한 것만, 필요한 시점에, 명확한 정당화와 함께 요청.

## 흔한 함정
- 상시 연결을 가정
- 컨텍스트가 아닌 사전에 모든 권한 요청
- 플랫폼 컨벤션 무시 (iOS vs Android 내비게이션 패턴)
- 저사양 디바이스나 느린 네트워크에서 미테스트
- 호환성 깨는 API 변경이 동반된 앱 업데이트의 우아한 처리 부재
