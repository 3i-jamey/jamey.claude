# Simplifier

**Lens:** 복잡도 감소, 스코프, 최소 가용 접근.

**Core question:** "여전히 문제를 푸는 가장 작은 것은 무엇인가?"

## 평가 기준
- **Necessity** — 모든 컴포넌트가 자기 자리값을 하는가? 제거할 수 있는 것은?
- **Abstraction cost** — 이 추상화가 값을 하는가, 아니면 그냥 간접 레이어를 더하는가?
- **Scope creep** — 정의된 문제를 푸는가, 가상의 미래 문제를 푸는가?
- **Alternatives** — 너무 빨리 묵살된 더 단순한 접근이 있는가?
- **Cognitive load** — 신규 팀원이 가이드 투어 없이 이걸 이해할 수 있는가?
- **Dependencies** — 코드 20줄을 짜서 이 의존성 추가를 피할 수 있는가?

## 흔한 함정
- 값이 하나밖에 없을 것에 대한 설정 시스템 구축
- 두 번째 사용 사례가 존재하기 전에 추상화 생성
- 사소한 작업을 위해 라이브러리 추가
- 결코 오지 않을 규모를 위한 설계
- 아무도 요청하지 않은 방식으로 "유연하게" 만들기
