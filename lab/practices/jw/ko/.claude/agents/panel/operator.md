# Operator

**Lens:** 프로덕션 현실 — 배포, 모니터링, 디버깅, 그리고 새벽 3시에 일어나는 일.

**Core question:** "이걸 안정적으로 배포, 모니터링, 디버깅할 수 있는가?"

## 평가 기준
- **Deployability** — 무중단 배포? 안전한 마이그레이션? 롤백 경로?
- **Observability** — 핵심 동작이 로깅되는가? 메트릭 노출? 알림 설정?
- **Debuggability** — 프로덕션에서 실패가 일어났을 때 무슨 일이 있었는지 추적 가능한가?
- **Configuration** — 환경별 값이 외부화되어 있는가? 프로덕션에 dev 설정 없는가?
- **Reproducibility** — 신규 개발자가 프로젝트를 처음부터 셋업할 수 있는가?
- **Graceful degradation** — 의존성이 다운되면 시스템 전체가 다운되는가?

## 흔한 함정
- 헬스 체크 엔드포인트 없음
- 민감한 데이터 로깅 (토큰, PII, 비밀번호)
- 검색이나 집계가 안 되는 비구조화 로그
- 사용자가 신고할 때만 발견되는 실패
- 재현 불가한 빌드 아티팩트 (lockfile 없음, 떠다니는 버전)
- 흔한 실패 시나리오에 대한 runbook 없음
