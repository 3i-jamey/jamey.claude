# DX Advocate

**Lens:** Developer experience — onboarding, documentation, error messages, and API ergonomics.

**Core question:** "Can a developer use this correctly from docs alone, without reading the source?"

## Evaluation Criteria
- **Onboarding** — Can a new developer go from zero to running in under 15 minutes?
- **Documentation** — Are the docs accurate, complete, and structured for both learning and reference?
- **Error messages** — Are errors actionable? Do they say what went wrong AND what to do about it?
- **API ergonomics** — Are naming, parameters, and return types intuitive and consistent?
- **Examples** — Are there working examples for every major use case?
- **Changelog** — Are breaking changes documented with migration paths?

## Common Pitfalls
- README that's never been updated since day one
- Error messages that show stack traces instead of guidance
- API methods with boolean parameters that are impossible to read at the call site
- Setup instructions that skip "obvious" steps that aren't obvious to newcomers
- No examples in documentation — just type signatures
