# 로컬 리뷰: $ARGUMENTS

## 워크플로우

1. 리뷰 대상 결정:
   - `$ARGUMENTS`에 파일/경로가 있으면 그것만 리뷰
   - 없으면 미커밋 전체: `git diff HEAD` + `git status`의 미추적 파일
   - 워킹 트리가 깨끗하면 최근 커밋 폴백: `git show HEAD`
2. 변경된 파일을 전체 읽습니다 — diff만으로는 import, 의존 방향 등 주변 컨텍스트를 놓칩니다.
3. `CLAUDE.md` / `CONTRIBUTING.md`에 명시된 아키텍처·코딩 규칙을 체크리스트에 합쳐 적용합니다.
4. 아래 체크리스트(이 파일 하단)를 수행하고 심각도별로 묶어 출력 포맷대로 보고합니다.

## 반드시 지킬 규칙

- 발견만 하고 자동으로 고치지 않는다 — 무엇을 반영할지는 사용자가 결정합니다. review 스킬의 정체성.
- 변경이 없으면 그대로 보고하고 이슈를 지어내지 않는다 — 거짓 양성은 신뢰를 떨어뜨려 다음 리뷰가 무시됩니다.
- `CLAUDE.md` / `CONTRIBUTING.md` 명시 규칙 위반은 항상 **HIGH**로 표기한다 — 팀이 합의한 규칙은 개인 취향보다 우선합니다.
- diff만 보지 않고 변경된 파일 전체를 읽는다 — diff는 주변 import·의존 방향·라이프사이클 컨텍스트를 보여주지 못합니다.

## 기본 규칙

- submodule pointer 변경이 들어 있으면 별도로 표시한다 — 일반 코드 변경과 섞이면 머지 후 빌드가 깨질 수 있습니다.
- 심각도는 HIGH/MED/LOW 셋만 사용한다 — 더 잘게 쪼개면 사용자가 우선순위를 잡기 어렵습니다.

## 체크리스트

**잔여물:**
- [ ] 이번 변경에 추가된 디버그 로그(`Log.*`, `console.log`, `print`, `dbg!`, `debugger`, `Timber` 임시), `TODO/FIXME` 없음
- [ ] 주석 처리된 코드 블록 없음
- [ ] 테스트에 `@Ignore`/`.only`/`.skip`/하드코딩된 인증정보 없음

**비밀값/설정:**
- [ ] diff에 API 키, 토큰, 비밀번호, 내부 URL 없음
- [ ] `.env`, `local.properties`, 키스토어/`*.pem`/`*.key` 같은 인증 파일이 스테이징되지 않음
- [ ] 새 환경변수·플레이버 설정은 팀이 볼 수 있는 곳에 문서화됨

**프로젝트 규칙 (CLAUDE.md / CONTRIBUTING.md):**
- [ ] 명시된 아키텍처 위치 규칙(레이어/모듈 위치) 준수
- [ ] 명시된 비동기/동시성 규칙 준수
- [ ] 명시된 null 안전성/타입 안전성 규칙 준수
- [ ] 명시된 라이프사이클/자원 관리 규칙 준수
- [ ] 명시된 의존 방향(예: `presentation → domain → data`) 위반 없음
- [ ] 명시된 베이스 클래스/패턴 사용

**정확성:**
- [ ] 리네임/삭제 전파됨 — 삭제된 심볼·문자열 리소스 참조 없음
- [ ] import 깨끗함 (미사용 없음, 누락 없음, wildcard 남용 없음)
- [ ] null이 될 수 있는 LiveData/StateFlow/Optional 경로 처리됨
- [ ] 에러 처리는 시스템 경계(API, DB, BLE, 카메라 등)에서만 — 내부 에러 삼키지 않음
- [ ] UI 프레임워크 패턴(Compose `remember`/state hoisting, MVI 상태 전이 등) 일관 사용

**범위:**
- [ ] 모든 hunk가 해당 티켓·작업 의도에 기여 — 곁다리 리팩토링 없음
- [ ] 호출 지점이 하나뿐인 추상화를 새로 만들지 않음
- [ ] 투기적 기능/플래그/설정 노브 없음
- [ ] 기존 SDK/모듈 인터페이스 깨지 않음

**테스트:**
- [ ] 동작 변경에 테스트 변경 따라옴 (UseCase, ViewModel 단위 테스트 우선)
- [ ] 코드 실행만 확인하는 테스트가 아니라 실제 동작·상태 전이를 검증
- [ ] 가능하면 로컬 테스트 스위트 실행 결과 확인

**문자열/리소스:**
- [ ] 새 사용자 노출 문자열은 리소스 파일(`strings.xml` 등)에 있음 (하드코딩 금지)
- [ ] 다국어 키가 한쪽 언어에만 들어가지 않음

## 출력 포맷

```
## 로컬 리뷰

### 판정: [PASS / PASS WITH FLAGS / FAIL]

### 이슈
- **[HIGH/MED/LOW]** <파일>:<줄> — <무엇이 잘못됐고 왜 중요한지>

### 통과
- 깔끔하게 통과한 체크 요약

### 제안
- 선택적, 블로커 아님
```
