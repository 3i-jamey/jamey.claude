/*
==========================================================================
 SKILL.md 작성 가이드 — 이 파일은 모든 스킬의 기준 템플릿이다.

 [사용법]
   1. 이 파일을 새 스킬 폴더에 복사: skills/<skill-id>/SKILL.md
   2. frontmatter와 각 섹션을 채운다.
   3. 비어 있는 선택 섹션은 통째로 삭제한다.
   4. 모든 주석(이 블록 포함)을 지우고 커밋한다.

 [주석 컨벤션]
   // 한 줄 주석 — 짧은 안내
   /* 여러 줄 주석 */

   HTML 주석(<!-- -->)을 쓰지 않는 이유:
   - 이 파일은 raw 텍스트로 편집·소비되는 일이 많아 "숨김"이 손해.
   - // /* */ 가 본문 markdown과 시각적으로 더 강하게 구분됨.
   - LLM은 어차피 모두 텍스트로 읽으므로 숨김 효과는 무의미하고,
     사람이 raw로 편집할 때 코드 스타일이 가독성에서 유리.

 [규칙 섹션 — 강제도로 2단 분리]
   LLM은 평면 bullet에서 위계를 잃기 쉬우므로 섹션으로 가른다.

   ## 반드시 지킬 규칙   위반 시 작업 중단. 안전·정합성·핵심 컨벤션·파괴적 행동 금지.
   ## 기본 규칙          명시적 사유 없으면 따르는 기본 동작.

   각 섹션 안에서 긍정/부정은 bullet 문장으로 표현:
   - "X 한다 — 왜"
   - "X 하지 않는다 — 왜"

   이유(— 왜)는 한 줄로 짧게. 같은 강제도 안에서 짝이 되는 규칙은 인접 배치.

 [frontmatter 필드]
   - name: 슬래시 커맨드명과 일치. kebab-case + 점 허용 (e.g., git.commit)
   - description: 호출 트리거의 핵심. 「무엇을, 언제, 왜」 한 문장.
                  너무 일반적이면 호출 안 되고, 너무 좁으면 충돌.
   - argument-hint: 입력 자리표시자. 인자 받지 않으면 생략 가능.
   - model: 스킬 실행 시 사용할 모델. **기본값은 sonnet**.
            값: sonnet / opus / haiku / inherit (현재 세션 모델 유지).
            해당 턴에만 적용되고 끝나면 세션 모델로 자동 복귀.
            바꿀 이유가 명확할 때만 변경(예: 무거운 점검 → opus, 짧은 작업 → haiku).

 [섹션 구조 — 코어 4 + 추가 섹션]
   모든 스킬은 다음 4개를 코어로 가정한다. 순서도 이 순서를 따른다.

   # 제목 + 컨텍스트 단락
   ## 워크플로우
   ## 반드시 지킬 규칙
   ## 기본 규칙

   필요할 때만 코어 4개 아래에 추가 섹션을 붙인다. 코어 사이에 끼우지 않는다.
   추가 섹션 권장 순서:

   ## 체크리스트            보고형 스킬에서 워크플로우 한 단계가
                            "아래 항목을 점검한다"가 될 때 분리.
   ## 출력 포맷             보고형 스킬(review/audit 등)에서 결과 구조 명시.
   ## 예시                 추상 규칙만으로 모호한 부분에만.
   ## 참고                 외부 자료가 있을 때만.

   위 목록 외 섹션도 필요하면 자유롭게 추가 가능.
   기준: "이 섹션이 없으면 스킬이 못 굴러가는가?" — 답이 yes면 추가.
   불필요한 섹션은 만들지 않는다. 코어 4개도 정말 비면 비워둔다(채우려 짜내지 않음).

 [공통 작성 원칙]
   - 명령은 코드 블록으로 박는다 — 자연어 설명만 있으면 LLM이 추측.
   - 한 단계 = 한 가지 일. 병렬 가능한 묶음만 "병렬로 실행:"으로 명시.
   - 본문에서 같은 규칙을 반복하지 말 것 — 한 번이면 충분.
   - description과 워크플로우 첫 줄이 중복되지 않도록.
==========================================================================
*/
