# Codex Agent Template

이 파일은 `home/dot-codex/agents/<agent-id>/AGENT.md`에 둘 에이전트 지침의 기준 템플릿입니다. 실제 에이전트를 만들 때는 placeholder와 안내 문장을 제거하고, 해당 역할에 필요한 내용만 남깁니다.

## Frontmatter

```yaml
---
name: <agent-id>
description: <Codex가 이 에이전트 지침을 언제 참고해야 하는지 한 문장으로 설명>
---
```

## 본문 구조

```markdown
# Agent: <agent-id>

**역할:** <이 에이전트가 책임지는 일>
**범위:** <다루는 작업과 다루지 않는 작업>

<입력을 어떻게 해석하고 어떤 결과를 반환하는지 1-3문장.>

## 워크플로우

1. <상태 확인 또는 입력 검증>
2. <컨텍스트 수집>
3. <실행 또는 관련 스킬 사용>
4. <검증과 보고>

## 반드시 지킬 규칙

- <위반하면 중단해야 하는 안전/정합성 규칙>
- <사용자 확인이 필요한 파괴적 작업>

## 기본 규칙

- <명시적 사유가 없으면 따르는 기본 동작>
- <프로젝트 관례와 충돌할 때의 우선순위>

## 출력 원칙

- <결론, 근거, 다음 단계 등 보고 구조>
```

## 작성 원칙

- Claude 전용 호출법(`@agent-*`)이나 `model/tools/context` frontmatter를 쓰지 않는다.
- 실제 작업 절차가 이미 스킬에 있으면 `$skill-name`으로 연결하고 같은 규칙을 중복하지 않는다.
- 프로젝트 지침은 `AGENTS.md`와 `CONTRIBUTING.md`를 우선한다.
- 저장소 변경, 커밋, push, 외부 서비스 호출은 사용자 의도와 현재 권한을 확인한 뒤 수행한다.
