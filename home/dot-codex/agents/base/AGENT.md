# Codex Agent 템플릿

이 파일은 새 에이전트 지침을 만들 때 복사해서 쓰는 기준 템플릿입니다.
Codex 환경에서는 모델명, 도구 허용 목록, fork 설정을 Claude frontmatter처럼 선언하지 않고, 필요한 동작과 제약을 본문 규칙으로 명확히 적습니다.

## 작성 절차

1. 이 디렉터리를 복사해 `agents/<agent-id>/AGENT.md`를 만든다.
2. frontmatter의 `name`을 실제 에이전트 이름으로 바꾼다.
3. `description`에 어떤 요청을 언제 이 에이전트에 맡겨야 하는지 적는다.
4. 본문에는 역할, 워크플로우, 반드시 지킬 규칙, 기본 규칙만 남긴다.
5. 새 에이전트에 필요하지 않은 섹션은 삭제한다.

## Frontmatter 규칙

- `name`과 `description`만 둔다.
- `name`은 kebab-case로 쓴다.
- `description`은 위임 트리거가 되므로 너무 일반적으로 쓰지 않는다.
- `model`, `tools`, `context` 같은 Claude 전용 필드는 쓰지 않는다.
- 도구 제한, 검증 명령, 보고 형식은 본문에 적는다.

## 본문 구조

```markdown
# <자연어 제목>

<역할, 범위, 사용자 요청을 해석하는 기준을 1~3문장으로 설명한다.>

## 워크플로우

1. <입력과 현재 상태를 확인한다.>
2. <필요한 조사나 작업을 수행한다.>
3. <검증한다.>
4. <결과를 보고한다.>

## 반드시 지킬 규칙

- <위반 시 작업을 멈춰야 하는 규칙> — <짧은 이유>
- <사용자 확인 없이 하면 안 되는 행동> — <짧은 이유>

## 기본 규칙

- <명시적 사유 없으면 따르는 기본 동작> — <짧은 이유>
```

필요할 때만 추가:

- `## 체크리스트`: 리뷰나 감사처럼 항목별 점검이 중요한 경우
- `## 출력 포맷`: 결과 구조가 고정되어야 하는 경우
- `## 예시`: 추상 규칙만으로 오해 가능성이 큰 경우
- `## 참고`: 외부 문서나 프로젝트 관례가 필요한 경우

## Codex 표현 규칙

- 파일 검색은 `rg` 또는 `rg --files`를 우선한다.
- 파일 읽기/수정/쓰기처럼 Codex가 수행할 행동을 자연어로 적고, Claude 전용 도구명에 의존하지 않는다.
- 프로젝트 규칙은 `AGENTS.md`와 `CONTRIBUTING.md`를 기준으로 삼는다.
- 파괴적 git 명령, 대량 삭제, force push처럼 되돌리기 어려운 행동은 사용자 확인 조건을 본문에 명시한다.
- 에이전트가 실제로 맡는 책임과 스킬로 위임해야 하는 책임을 분리해서 쓴다.

## 작성 원칙

- 한 단계에는 한 가지 일만 둔다.
- 병렬 가능한 확인은 "병렬로 실행"처럼 명시한다.
- 같은 규칙을 여러 섹션에 반복하지 않는다.
- 출력 포맷이 있으면 코드블록으로 정확히 정의한다.
- 보고는 결론, 근거, 다음 단계 순서로 짧게 유지한다.
