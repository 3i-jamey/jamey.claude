# 컨텍스트 분석: $ARGUMENTS

이 스킬은 **현재 대화의 맥락을 한·영 양국어로 요약**해 출력하는 **순수 분석기**다. 카테고리(결정/작업/이슈)로 사실을 잘게 쪼개지 않고, 세션이 어떤 흐름이었는지 *맥락 그대로* 산문으로 풀어쓴다 — 미래의 사용자(또는 다른 LLM)가 이 요약만 읽고도 세션의 모양을 재구성할 수 있어야 한다.

파일을 쓰거나 외부 시스템에 보내지 않는다 — 호출자(다른 스킬, 또는 사용자 지시를 받은 main 에이전트)가 자기 포맷(md 파일, Jira ADF, Slack 메시지 등)으로 변환해 사용한다.

## 워크플로우

1. (인자 받지 않음) 현재 대화 컨텍스트를 입력으로 본다. `$ARGUMENTS`가 들어오면 분석 초점을 좁히는 힌트로만 사용 (예: "translator 스킬 결정만").
2. **주제** 한 줄: 이 세션이 무엇에 관한 것인지 한국어·영어 각각 한 문장.
3. **한 줄 요약**: 세션의 핵심(문제·결정·결과)을 한국어·영어 각각 한 문장으로 압축.
4. **내용**: 세션의 흐름·의사결정·작업·발견·열린 이슈를 자유 형식으로 풀어쓴다. 어떻게 시작했고 어떤 분기점들을 지나 어디로 갔는지가 자연스럽게 읽혀야 한다. 단락·bullet·코드블록 자유롭게 섞어 쓴다 — 단, 형식 자체가 목적이 되지 않게.
5. 한국어 블록 먼저, 그 다음 영어 블록. 두 블록은 같은 정보를 같은 흐름으로 담는다.
6. 리턴 형식대로 출력 후 종료.

## 강제 규칙(!) : !는 IMPORTANT와 동일

- 한국어 블록(`## 한글(ko)`) → 영어 블록(`## English(en)`) 순. 인터리브 금지.
- 두 블록은 동일한 정보를 동일한 흐름으로 담는다. 한쪽에만 있는 항목 금지.
- `결정/작업/열린 이슈` 같은 고정 카테고리로 항목을 분리하지 않는다 — 맥락을 끊어 사실 나열로 만들지 말 것.
- 사용자 메시지의 비밀값/토큰/API 키처럼 보이는 문자열은 `<REDACTED>`로 마스킹.
- 대화에서 실제로 일어난 것만 적는다 — 추측·일반 조언·"좋은 관행" 일반론 금지.
- 파일을 쓰지 않는다, MCP를 호출하지 않는다, 외부 시스템에 보내지 않는다 — 순수 분석기.
- raw transcript·tool 호출 verbatim·시스템 reminder를 옮겨 적지 않는다.
- 리턴 형식 외 추가 설명·헤더·푸터를 붙이지 않는다.

## 기본 규칙

- **내용** 섹션은 산문이 기본. 정말 나열이 더 명확할 때만 bullet 사용. 코드 변경이 본질이면 코드블록 인용.
- 톤: 객관적·간결. 1인칭 회고("우리가 ~했다") 대신 명사형·서술체("~결정", "~로 전환됨").
- 분량: 내용 섹션은 보통 1~4 단락 또는 12~20줄. 너무 짧으면 맥락 손실, 너무 길면 노이즈.
- 파일 경로·명령·식별자는 인라인 코드(`` ` ``)로 표시.
- "왜"가 비자명할 때만 명시. 자명한 이유는 생략.

## 마무리 규칙

- 검증: 분석할 컨텍스트가 사실상 비어 있음(인사·잡담만) → `BLOCKED: EMPTY_CONTEXT`로 종료.
- 보고: 중간 추론·후보 항목 검토 과정은 출력하지 않는다.
- 결과: 리턴 형식 그대로만 출력. 호출자가 파싱해서 사용.

## 리턴 형식

```markdown
## 한글(ko)

### 주제
<한국어 핵심 주제 한 줄>

### 한 줄 요약
<세션의 핵심을 한 문장으로>

### 내용
<자유 형식 산문 — 흐름·의사결정·작업·발견·남은 이슈가 자연스럽게 읽히도록.
필요할 때 bullet·코드블록 섞어 사용. 형식이 목적이 되지 않게.>

## English(en)

### Topic
<one-line English topic>

### Summary
<one-sentence essence>

### Content
<free-form prose mirroring the ko block — same flow, same level of detail.>
```

호출자는 `## 한글(ko)` / `## English(en)` 블록을 분리하고, 각 블록의 `### 주제`/`### Topic` (제목용), `### 한 줄 요약`/`### Summary` (paragraph용), `### 내용`/`### Content` (본문용) 세 부분을 자기 포맷으로 매핑한다.

## 호출 예시

- **사용자가 직접:** "이 컨텍스트 분석해줘" → analyzer-context 호출 → main 에이전트가 출력을 받아 그대로 노출하거나, 사용자가 "이걸 `notes/today.md`에 저장" 요청 시 main 에이전트가 ad-hoc으로 frontmatter 붙여 Write.
- **다른 스킬에서:** `jira-comment-context`가 `Skill(analyzer-context)` 호출 → 주제/요약/내용을 받아 ADF heading + paragraph + expand(내용 markdown → ADF 노드 변환) 구조로 조립 → MCP로 댓글 등록.
