# 계획·설계 그릴: $ARGUMENTS

사용자가 가져온 계획·설계를 끝까지 파고들어 공유된 이해에 도달하기 위한 인터뷰 스킬. 한 번에 한 질문씩, 의사결정 트리의 모든 갈래를 의존성 순서대로 해결한다. 각 질문엔 추천 답변을 함께 제시한다.

## 워크플로우

1. 그릴 대상의 범위·목표를 한 줄로 정리해 사용자에게 확인받는다.
2. 결정해야 할 갈래를 식별하고 의존성 순서로 정렬한다.
3. 가장 우선되는 미해결 갈래에 대해 한 가지 질문을 던진다. 추천 답·근거를 함께 제시한다.
4. 사용자 답을 받아 결정 트리에 반영하고, 다음 갈래로 이동한다 (3번 반복).
5. 모든 갈래가 해결되면 합의된 결정 요약을 출력하고 종료한다.

## 반드시 지킬 규칙

- 질문은 한 번에 하나씩 던진다.
- 각 질문엔 추천 답변과 짧은 근거를 함께 제시한다.
- 코드베이스를 읽으면 답이 나오는 질문은 사용자에게 묻지 않고 직접 탐색한다.
- 미해결 갈래가 남은 채로 합의 요약을 내보내지 않는다.

## 마무리 규칙

- 모든 갈래의 결정 사항을 표로 요약해 출력한다.
- 미해결 갈래가 남았는데 사용자가 중단을 요청했다면 `BLOCKED: <남은 갈래>`로 종료한다.

## 리턴 형식

```
Q. <한 문장 질문>
추천: <한 문장 추천 답>
근거: <1~2문장>

계속하려면 답을, 멈추려면 "stop"을 입력해주세요.
```

사용자 답을 받으면 결정 트리에 반영한 뒤 다음 미해결 갈래로 이동한다.
